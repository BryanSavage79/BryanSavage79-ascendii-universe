## How to Contribute
Welcome! Your contributions are what make this project thrive. Here’s how you can get involved and help us grow together.

1. **Understand the Vision**  
   - Review the [Manifesto](./MANIFESTO.md) to get familiar with our community values, circulation mindset, and project goals.  
   - Explore the core components in this repository and the conceptual framework behind **Nexus of Equity**.

2. **Find a Starting Point**  
   - Browse the **[Issues]** tab to see current tasks, feature requests, and bug reports.  
   - Check out labels like `good first issue` or `help wanted` for beginner-friendly opportunities.  
   - Propose new ideas if you see unmet opportunities aligned with our vision.

3. **Get Set Up**  
   - **Fork** this repository and **clone** it locally.  
   - Follow the **[Development Setup Guide]** for environment configuration and dependencies.  
   - Run existing tests to verify your setup is working.

4. **Make Your Contribution**  
   - Work on your feature or fix in a **feature branch** (`git checkout -b feat/your-feature`).  
   - Follow our **[Code Style Guidelines]** to maintain consistency.  
   - Write clear commit messages explaining the **“why”** behind changes.

5. **Submit a Pull Request**  
   - Create a pull request (PR) referencing relevant issues.  
   - Provide a description of your changes and how they support the project vision.  
   - Engage in any requested revisions through collaborative review.

6. **Engage and Collaborate**  
   - Join our community channels:  
     - **[Discord]** • **[Forum]** • **[X/Twitter]**  
   - Attend periodic meetings or calls to stay aligned with ongoing developments.

7. **Recognition and Rewards**  
   - Contributions are recognized through **leaderboards**, **titles**, **influence**, and **creative co-ownership** as outlined in the Manifesto.  
   - See the **[Rewards Table]** below for how impact translates into roles and responsibilities.

### How to Create an Issue
1. Click **Issues → New issue**.  
2. Choose a template (or create a custom one).  
3. Use clear titles and the following labels:  

   - `good first issue` – perfect for newcomers  
   - `help wanted` – needs external expertise  
   - `bug`, `feature`, `documentation`, `enhancement`
